## search-index.csv 数据归一化优化

### 方案
- domain/field/subfield 在 search-index.csv 中以短 ID（D1/F1/S1）替代文本
- 新增 `public/data/lookup.csv`（type,id,name 格式），前端加载后还原名称
- 4 个 domain → D1-D4, 26 个 field → F1-F26, 244 个 subfield → S1-S244
- 前端 loadData() 并行加载 lookup.csv，解析后构建 lookupMap 对象
- tree-skeleton.json 中的节点 ID 也使用归一化短 ID

### 压缩效果
- search-index.csv: 488KB → 271KB（-44.4%，节省 217KB）
- lookup.csv: 9.7KB（新增，274 条映射记录）
- 净节省约 207KB

### 边界情况处理
- 8 个 subfield 同名但属于不同 field（如 Genetics 同时属于 Medicine 和 Biochemistry）
- lookup.csv 按唯一名称存储（244 条），field 上下文由 search-index 的 fid 列提供
- tree-skeleton 中同一 subfield 在不同 field 下会出现两次（252 个 subfield 节点 vs 244 个唯一名称）

### 构建脚本更新
- `scripts/build-frontend-data.js` 新增 buildLookupTables() 和 buildLookupCSV() 函数

## Concepts 前端修复：独立路由 + 6 级数据展示

### 问题
1. OpenAlex Concepts API 不返回 ancestors 字段（均为 null），无法构建父子树
2. build-frontend-data-concepts.js 只生成了 Level 0 (38个概念)，L1-L5 完全缺失
3. Topics 和 Concepts 无独立路由，切换时数据可能互相干扰

### 修复
1. **重构 build-frontend-data-concepts.js**：生成全部 6 级 CSV (level0~level5.csv)
   - 树骨架包含 6 个 domain 节点 (L0-L5)，每个带 _concept_file 懒加载
   - 各级数据量: L0(38), L1(568), L2(27605), L3(27640), L4(12994), L5(6181)
   - 总计 75026 个概念，6 个 CSV 文件 (6MB)
2. **前端 hash 路由**：使用 `window.location.hash` (#topics / #concepts)
   - 切换时通过 hash 变化触发，支持浏览器前进/后退
   - Topics 和 Concepts 数据各自只加载一次，互不干扰
3. **Concepts 搜索**：使用 concepts/search-index.csv，搜索路径 [levelGroupName, conceptName]
   - 搜索结果限制 200 条，避免加载过大 CSV
4. **Concepts 层级说明**：Info card 中显示 L0~L5 分级描述

### 关键文件
- `scripts/build-frontend-data-concepts.js` — 重构后生成 6 级数据
- `index.html` — hash 路由 + 独立数据加载 + concepts 搜索

- `public/data/concepts/concepts/level0~5.csv` — 6 个懒加载 CSV

## Concepts 前端修复：独立路由 + 懒加载搜索 + 分页

### 问题
1. loadConcepts() 和 loadTopics() 一起加载 tree skeleton + search-index.csv，search-index.csv (2.6MB/75026行) 阻塞树渲染
2. Concepts 页面完全看不到数据——因为加载慢 + 解析 75026 行阻塞主线程
3. 概念 Level 2/3 有 27000+ 行，全部渲染到 DOM 导致页面卡顿

### 修复
1. **拆分数据加载为两阶段**：
   - Phase 1: `loadTopicsTree()` / `loadConceptsTree()` 只加载 tree skeleton (<2KB)，立即渲染树
   - Phase 2: `ensureSearchData()` 搜索时按需加载 search-index.csv，首次搜索时显示 "Loading search index..."
2. **概念默认展开级别降低**：topics `defaultExpandLevel=2`，concepts `defaultExpandLevel=1`
   - Concepts 只展开 Level 节点标题，不自动加载大 CSV
3. **分页渲染**：大概念列表先显示 100 个，点击 "Show more" 加载更多
4. **构建流程修复**：`npm run build` 改为直接复制 index.html + public/ 到 dist/
   - 旧流程 `tsc && vite build` 会丢失 index.html 内嵌代码
   - 新脚本 `scripts/build-copy.cjs` 直接复制，无需 Vite 打包

### 关键改动
- `index.html`: loadTopics→loadTopicsTree+ensureTopicsSearchData, loadConcepts→loadConceptsTree+ensureConceptsSearchData
- `index.html`: searchTopics 改为 async, 首次搜索 await ensureSearchData()
- `index.html`: TreeNodeComponent 新增 visibleCount 分页状态
- `package.json`: build 脚本改为 build-data + build-data:concepts + build:copy
- 新增 `scripts/build-copy.cjs`: 复制 index.html + public/ → dist/
